# ego_single_goal parameter tuning

Launch:

```bash
roslaunch ego_single_goal ego_single_goal_px4.launch key:=value
```

## Must Keep

These parameters should normally stay fixed for the structured Task A scene:

| Parameter | Default | Meaning |
| --- | ---: | --- |
| `single_goal_only` | `true` | Stop after the first Task A goal instead of switching to transfer/Task D. |
| `guide_path_topic` | `/competition/guide_path` | Final corridor path consumed by `autonomous_flight`. |
| `task_a_guide_path_topic` | `/task_a/guide_path` | Corridor generated by wall-cylinder gap detection. |
| `marker_topic` | `/task_a/markers` | RViz markers for wall, cylinders, candidate gaps, and selected gap. |
| `forward_axis_x/y` | `0.0/-1.0` | Task A structure matching / guide-path direction. Front-cloud ROI can instead use odom yaw. |
| `lateral_axis_x/y` | `1.0/0.0` | Task A lateral direction, currently hardcoded in launch node params. |

## Scene Target

| Parameter | Default | Effect | Tune When |
| --- | ---: | --- | --- |
| `goal_x` | `3.120975971221924` | Final x after passing the selected wall-cylinder gap. | Goal is misplaced in map frame. |
| `goal_y` | `-5.956602573394775` | Final y after passing the selected wall-cylinder gap. | Goal is misplaced in map frame. |
| `goal_z` | `1.0` | Flight/goal height. | Vehicle should pass higher/lower. |
| `task_a_reach_radius` | `0.35` | Distance threshold to enter hover at final goal. | Increase if the vehicle oscillates near goal; decrease for stricter arrival. |

## Detection And Structure Matching

| Parameter | Default | Effect | Tune Direction |
| --- | ---: | --- | --- |
| `task_a_safe_radius` | `0.35` | Minimum half-width clearance used to decide whether a gap is passable; aligned with the inflated 0.5m robot footprint. | Increase for more clearance; decrease only if valid gaps are rejected. |
| `task_a_cylinder_forward_window` | `0.65` | Allows cylinders to match the wall if their forward coordinate is within this window. | Increase if cylinders are detected but not paired with wall; decrease if wrong cylinders are matched. |
| `task_a_predicted_gap_width` | `1.0` | Expected wall-cylinder gap width used in candidate scoring. | Set near real scene gap width. |
| `task_a_use_odom_front_roi` | `true` | Filters detection input to the UAV-front ROI using `/mavros/local_position/odom` yaw. | Set false to use the fixed Task A forward axis. |
| `task_a_odom_timeout` | `1.0` | Max age for odom before falling back to fixed-axis ROI. | Increase only if odom timestamps lag but remain valid. |
| `task_a_front_yaw_offset` | `0.0` | Extra yaw offset in radians applied to the odom-front ROI. | Use if MAVROS yaw and physical LiDAR/front direction differ. |
| `task_a_wall_length` | `1.0` | Expected wall length used to score/select wall candidates. | Set near measured wall length. |
| `task_a_wall_width` | `0.50` | Expected wall thickness/depth used to score/select wall candidates. | Set near measured wall width. |
| `task_a_wall_infer_height_margin` | `0.05` | Extra z margin added around the measured wall face height when sizing from point cloud. | Increase if the visible wall top/bottom is slightly clipped; decrease if height looks too tall. |
| `task_a_cylinder_diameter` | `0.40` | Expected cylinder diameter used to score/select cylinder candidates. | Set near measured cylinder diameter. |
| `task_a_obstacle_height` | `1.0` | Expected wall/cylinder height used to score candidates. | Set near visible structure height. |
| `task_a_min_structure_height` | `0.45` | Minimum observed point-cloud height for wall/cylinder candidates. | Lower if only the lower/middle part is visible; raise to reject clutter. |
| `task_a_lock_detection` | `true` | Multi-frame lock before publishing corridor. | Set false only for debugging unstable point cloud. |
| `task_a_saved_prior_prefer_roi` | `true` | When saved manual clicks exist, prefer the current-cloud clustering result inside that prior ROI before global auto detection. | Keep true after manual teaching; set false only to compare pure global detection. |
| `task_a_refine_duration` | `2.0` | Time window for stable structure refinement. | Increase for noisy clouds; decrease for faster startup. |
| `task_a_min_refine_matches` | `8` | Minimum valid observations before locking. | Increase for reliability; decrease for faster first lock. |
| `task_a_stable_center_threshold` | `0.12` | Allowed variation of selected gap center during refinement. | Increase if detection jitters slightly; decrease if lock is too loose. |
| `task_a_stable_gap_threshold` | `0.10` | Allowed variation of selected gap width during refinement. | Increase if measured width fluctuates; decrease for stricter lock. |
| `task_a_max_refine_duration` | `4.0` | Maximum time before refinement gives up waiting. | Increase if first lock takes too long under sparse point cloud. |

## Point Cloud And Map

| Parameter | Default | Effect | Tune Direction |
| --- | ---: | --- | --- |
| `raw_cloud_topic` | `/livox/lidar` | Input Livox point cloud topic. | Change to actual LiDAR topic on the NUC. |
| `cloud_topic` | `/world_cloud` | Transformed cloud used by detector and RViz. | Usually keep. |
| `cloud_downsample` | `1` | Downsampling in cloud transform bridge. | Increase if CPU load is high; keep low for small obstacles. |
| `task_a_forward_min/max` | `-0.20/9.0` | Front ROI distance window; with odom-front mode this is relative to the UAV. | Reduce max to ignore far clutter; raise min only if near-body points pollute. |
| `task_a_lateral_min/max` | `-5.0/7.0` | Front ROI lateral window; with odom-front mode this is relative to the UAV heading. | Narrow to reject side clutter. |
| `map_resolution` | `0.1` | Occupancy map resolution. | Smaller is more accurate but heavier; larger is faster but coarse. |
| `raycast_max_length` | `15.0` | Max raycast range for map update. | Reduce indoors for speed; increase for long-range mapping. |
| `local_update_range_x/y/z` | `8.0/8.0/5.0` | Local map update box around UAV. | Increase for faster flight; reduce for CPU. |
| `local_map_size_x/y/z` | `20.0/20.0/6.0` | Local map size. | Must cover detection and planning horizon. |
| `detector_range_x/y/z` | `20.0/20.0/5.0` | Onboard detector local sensor range. | Reduce if false positives far away; increase if objects are missed. |
| `detector_ground_height` | `0.2` | Ground filtering baseline. | Raise if ground points pollute obstacle clusters. |

## Planner And Control

| Parameter | Default | Effect | Tune Direction |
| --- | ---: | --- | --- |
| `max_vel` | `1.5` | Desired max flight velocity. | Lower for first NUC tests; increase after corridor is reliable. |
| `max_acc` | `1.0` | Desired max acceleration. | Lower for smoother flight; increase for aggressive response. |
| `desired_angular_velocity` | `1.2` | Yaw/angular speed limit. | Lower if yaw is unstable. |
| `planner_pose_target_lookahead` | `0.30` | Lookahead for planner pose target. | Increase for smoother trajectory; decrease for tighter reaction. |
| `control_target_lookahead` | `0.25` | Lookahead for control target. | Increase for smoother control; decrease for tighter tracking. |
| `publish_px4_raw_setpoint` | `false` | Publish raw PX4 setpoints to MAVROS. | Turn on only for PX4 offboard control tests. |
| `enable_control_output` | `false` | Enables controller output inside `autonomous_flight`. | Turn on only when control pipeline is ready and verified. |
| `auto_takeoff` | `false` | Auto takeoff behavior. | Keep false for manual staged testing. |

## Dynamic Replanning

| Parameter | Default | Effect | Tune Direction |
| --- | ---: | --- | --- |
| `replan_time_for_dynamic_obstacles` | `0.30` | Replan period when dynamic collision risk exists. | Lower for faster reaction; higher for CPU. |
| `dynamic_collision_check_step` | `0.01` | Time step for collision prediction. | Lower is safer but heavier. |
| `dynamic_collision_prediction_time` | `1.5` | Future time horizon for dynamic collision prediction. | Increase for faster flight; reduce if too conservative. |
| `dynamic_collision_safety_margin` | `0.25` | Extra clearance for dynamic obstacles. | Increase for safety; decrease if planner is too conservative. |
| `dynamic_collision_replan_distance` | `0.60` | Distance threshold to trigger replan. | Increase for earlier replanning. |

## B-spline And Polynomial Trajectory

| Parameter | Default | Effect | Tune Direction |
| --- | ---: | --- | --- |
| `bspline_min_height` | `0.7` | Minimum planned B-spline height. | Raise if vehicle flies too low. |
| `bspline_max_height` | `1.3` | Maximum planned B-spline height. | Raise if vertical clearance allows. |
| `bspline_prediction_horizon` | `2.0` | Planning horizon for B-spline dynamic obstacle handling. | Increase for smoothness; reduce for tight spaces. |
| `bspline_max_path_length` | `7.0` | Maximum B-spline path length. | Increase for longer corridors; reduce for local behavior. |
| `bspline_distance_threshold` | `0.40` | Clearance threshold for static obstacles. | Increase for safety; decrease if corridor is too narrow. |
| `bspline_distance_threshold_dynamic` | `0.30` | Clearance threshold for dynamic obstacles. | Increase for safety around moving objects. |
| `bspline_weight_feasibility` | `1.0` | Weight for dynamic feasibility. | Increase if trajectory violates speed/acc limits. |
| `bspline_weight_dynamic_obstacle` | `1.2` | Weight for dynamic obstacle avoidance. | Increase if too close to moving obstacles. |
| `poly_initial_radius` | `0.45` | Initial polynomial corridor radius. | Increase if corridor is too tight; decrease for narrow passage. |
| `poly_constraint_radius` | `0.45` | Polynomial corridor constraint radius. | Increase for safety; decrease for narrow passage. |

## Recommended First Test

```bash
roslaunch ego_single_goal ego_single_goal_px4.launch \
  rviz:=true \
  max_vel:=0.6 \
  max_acc:=0.5 \
  task_a_safe_radius:=0.35 \
  task_a_refine_duration:=2.5
```
